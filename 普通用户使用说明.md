# 普通用户使用说明

这个项目适合做成本地优先的微信聊天关系分析器：数据留在自己的电脑上，通过本地网页查看聊天记录、联系人和关系分析。

## 一键启动

推荐双击项目里的 `start_desktop.command`。

它会打开一个小控制窗口，可以直接：

- 打开聊天界面
- 打开年度总览
- 打开关系报告
- 打开回忆搜索
- 打开隐私安全中心
- 刷新聊天快照

如果你更喜欢只开网页，也可以双击 `start.command`。

启动器会自动完成这些事：

1. 检查本地是否已经有聊天快照。
2. 如果有快照，启动本地网页。
3. 自动打开浏览器。
4. 如果 `8765` 端口被占用，自动尝试后面的端口。

网页地址通常是：

```text
http://127.0.0.1:8765/
```

## 刷新聊天快照

如果已经提取过密钥，想把聊天记录更新到最新，可以在项目目录运行：

```bash
python3 launch.py --sync
```

或者继续使用开发者脚本：

```bash
./update.sh
```

## 第一次使用

第一次使用仍然需要完成一次密钥提取和数据库解密。这一步比较敏感，可能需要临时关闭 SIP，所以启动器不会自动替你做。

推荐流程：

```bash
brew install llvm sqlcipher
PYTHONPATH=$(lldb -P) python3 find_key_memscan.py
python3 launch.py --sync
```

完成后，之后日常打开只需要双击 `start.command`。

## 隐私边界

- 网页只绑定在 `127.0.0.1`，默认只能本机访问。
- 项目不会上传聊天记录。
- `wechat_keys.json`、`decrypted/`、`exported/` 都不应该提交到 GitHub。
- 不要用这个工具分析别人的设备或未经授权的数据。

## 核心页面

### 年度总览

年度总览会基于最近活跃的普通联系人生成关系地图，包括聊得最多的人、关系温度最高的人、连续互动最长的人、夜间聊天较多的人、月份分布和年度高频词。

第一版总览采用采样分析，不会一次性扫描所有联系人，因此速度更适合普通用户日常打开。它适合发现关系趋势，不等于完整全量统计。

### 聊天界面

左侧选择联系人，中间查看聊天记录，右侧查看基础关系分析。顶部可以选择日期范围，聊天记录和关系分析会一起刷新。

顶部会显示当前聊天快照的新鲜度。如果提示快照已经比较久，可以回到启动窗口点击“刷新聊天快照”。

### 回忆搜索

回忆搜索可以在所有本地聊天快照里查关键词，适合找地址、约定、文件名、合同、咖啡、旅行、某段争论等突然想起的内容。

搜索结果不会上传，会直接从本机解密后的数据库读取。点击某条结果，可以回到对应联系人的聊天界面。

### 关系报告

关系报告会把同一个日期范围内的数据整理成更完整的阅读页，包括关系温度、关系类型、互动占比、常聊时段、高频日期、消息构成和常见关键词。

新版报告还会做更深一层的社交关系解读：

- 互动密度：这段时间聊得是否集中
- 关系连续性：是否跨日期稳定延续
- 回应均衡：是不是一方明显更主动
- 对话轮转：有没有来回响应
- 时间边界：是否大量发生在夜间或周末
- 表达丰富度：文本表达是否足够丰富

这些判断只来自聊天频率、时间和文本长度等可解释指标，不等于真实亲密度。工作协作、线下关系、共同圈层、当时处境都会改变含义。

如果想保存报告，可以点击关系报告页右上角的“导出报告”。导出的是 Markdown 文件，数据不会上传。

### 隐私安全中心

隐私安全中心会检查本地访问状态、聊天快照大小、密钥文件状态、GitHub 发布前的敏感文件风险。它不会删除文件，只做提醒。

## 面向开源的定位

建议把项目定位成：

> 一个本地运行的 macOS 微信聊天记录分析工具，帮助用户在不上传数据的情况下，探索自己与好友的聊天历史、互动节奏和关系变化。

不要把项目宣传成破解工具。密钥提取只是为了读取用户自己的本机数据，真正的产品价值应该放在本地浏览、关系分析、时间线回顾、年度报告和隐私优先上。
